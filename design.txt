# Opus.Prism - Comprehensive Workflow & Architecture Design

This document outlines the entire workflow of the Opus.Prism codebase, detailing the models used, the problems solved, the task of each module, and provides a precise prompt (with Mermaid code) to generate a Draw.io file.

---

## 1. Problems Identified & Solved
The codebase (`persona-cut`) actively solves three primary host problems:

1. **Problem 1: Personalization in Generated Video:** 
   - *Problem:* A single story/script typically results in one generic video output.
   - *Solution:* **PersonaCut** acts as an AI compiler. It takes one base script and generates multiple variant packs (e.g., a 15s casual TikTok, a 45s technical YouTube short, a 30s professional LinkedIn post) without rerunning the heavy lifting generation steps from scratch.
2. **Problem 5: Critic in the Loop:**
   - *Problem:* Pipelines generate and ship without verifying quality, tone, or factual faithfulness.
   - *Solution:* Placed throughout the system. The **Scorecard** agent pre-evaluates generated variant packs (scoring out of 50). Later, the **TextGuard VLM** evaluates frame-by-frame OCR results to provide actionable verdicts (pass/fail/re-render).
3. **Problem 3: Accurate Text (No Gibberish):**
   - *Problem:* AI-generated videos often contain garbled or hallucinated text overlays.
   - *Solution:* **TextGuard QA** extracts frames using `ffmpeg`, runs an OCR engine against them, and compares the detected text against the expected captions from the Variant Pack. If text is wrong, it uses a VLM to diagnose the issue and generates a "patched" video.

---

## 2. Models & Technologies Used
The system is highly modular, utilizing specific models for specialized tasks:

- **Large Language Model (Variant Generation & Evaluation):** `Qwen/Qwen2.5-72B-Instruct`
  - *Usage:* Used by the CrewAI agentic pipeline (Extractor, Strategist, Writer, Planner, Critic) to understand the source script and generate complex JSON variant packs.
- **Large Language Model (OPBot Chat Assistant):** `Qwen/Qwen2.5-7B-Instruct`
  - *Usage:* Powers the context-aware chatbot on the frontend, using a smaller, faster model to guide the user.
- **Vision-Language Model (TextGuard QA Verdicts):** `Qwen/Qwen2.5-VL-7B-Instruct`
  - *Usage:* Inspects specific video frames where OCR failed, comparing the visual text to expectations and outputting JSON verdicts.
- **Optical Character Recognition (OCR):** `Qwen2.5-VL`
  - *Usage:* Frame-by-frame text extraction during the TextGuard QA phase.
- **Text-to-Speech (TTS):** `ElevenLabs` (Model: `eleven_monolingual_v1`, Voice ID: `21m00Tcm4TlvDq8ikWAM`)
  - *Usage:* Voice playback for OPBot chatbot responses.
- **Web Speech API:** Native browser speech-to-text for OPBot voice input.

## 3. Tasks of Each Module

### Module A: Frontend (Next.js + Tailwind)
- **Welcome Page (`/`):** Landing interface introducing the three pillars of Opus.Prism.
- **Project Creator (`/personacut`):** Accepts the source script/idea, style guide, and target audience configurations (platform, duration, tone).
- **Results Dashboard (`/results`):** Displays the generated Variant Packs (voiceover script, captions, scene-by-scene prompts, duration, and Critic Scorecard). Allows exporting JSON and copying detailed prompts for external renderers.
- **TextGuard QA (`/qa`):** File upload interface for rendered `.mp4` videos. Displays side-by-side original vs. patched videos and a frame-by-frame QA report.
- **OPBot (`ChatWidget.tsx`):** Ambient AI assistant that reads routing context (e.g., knows if you are on the QA page) to provide relevant actions.

### Module B: PersonaCut Backend (FastAPI + CrewAI) 
- **Agent 1 (Source Script Analyst):** Extracts immutable "Story Facts" (banned terms, required terms, numbers, core message) from the raw input script.
- **Agent 2 (Audience Strategist):** Maps the story facts to the specific demographic/platform parameters.
- **Agent 3 (Video Script Writer):** Crafts the actual voiceover and captions.
- **Agent 4 (Visual Storyboarder):** Translates the script into precise visual generative prompts (Scene start/end times).
- **Agent 5 (Quality Critic):** Grades the variant JSON out of 50. If the score is below threshold, it triggers a system retry.

### Module C: TextGuard QA Backend
- **Frame Extractor:** Slices the uploaded `.mp4` into JPEGs based on a target FPS.
- **OCR Engine:** Scans each frame for visible text (`Qwen2.5-VL`).
- **Matcher (`compare.py`):** Uses string similarity to compare detected text against the expected captions (inherited from PersonaCut).
- **VLM Inspector:** If string similarity is low, the frame is sent to `Qwen2.5-VL` to determine if the text is genuinely garbled, styled strangely, or hallucinated.
- **Patcher:** Redraws correct text boxes over the video to create a fixed `patched.mp4`.

## 4. Prompt to Generate the Draw.io File

*Instructions for the User:* You can automatically generate the visual workflow diagram inside Draw.io by using the Mermaid syntax below.
1. Open [Draw.io / diagram.net](https://app.diagrams.net/)
2. Click **Arrange** -> **Insert** -> **Advanced** -> **Mermaid...**
3. Paste the Mermaid code block below and click **Insert**.
